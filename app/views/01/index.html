If you haven’t already, visit this site’s home page for a brief introduction to the Arduino.

Objectives

  • Develop familiarity with the Arduino’s basic computer connections.
  • Understand how to configure the Arduino hardware to communicate with its software.

What You Need

  • The Arduino software, which is available for macOS, Windows, and Linux. Download it from https://www.arduino.cc/en/Main/Software#download.
  • The Arduino board (we’re using the UNO revision 3 in these tutorials). You can acquire an Arduino Uno — or compatible — board from places like the Arduino store, Adafruit, SparkFun, or Mouser.
  • A standard Type-A to Type-B usb cable.

Configuring the Arduino Board to Communicate with the Arduino Software

Burden: 2 minutes

Connecting the board to your computer does not indicate that a connection between the software and the hardware is functioning. There are two quick settings that must be configured in order for the board to communicate with the software: board and port, both found under the tools menu in the Arduino software. Note: Always check these two settings when switching out boards, changing computers, or even starting a new work session.

  1. Using your Type-A to Type-B usb cable, connect the Arduino to an available usb port on your computer.
  2. Launch the Arduino software.
  3. Go to the tools menu, hover your mouse over the board option, then check off arduino uno.
  4. Go back to the tools menu, hover your mouse over the port option, then check off the port on which your Arduino is connected. It may say, “Arduino Uno” to the right of the port on which your Arduino is connected.

The configuration is complete. Let’s now test it.

Testing Communication Between the Arduino Board and the Software

Burden: 3 minutes

If our configuration from the previous section works, then we should be able to upload a sketch (Arduino nomenclature for a program, or app) from our computer to the Arduino board.

  1. In the Arduino software, go to file | examples | 01.basics and choose blink. You should see a sketch that looks like the following: 
    /*
  Blink

  Turns an LED on for one second, then off for one second, repeatedly.

  Most Arduinos have an on-board LED you can control. On the UNO, MEGA and ZERO
  it is attached to digital pin 13, on MKR1000 on pin 6. LED_BUILTIN is set to
  the correct LED pin independent of which board is used.
  If you want to know what pin the on-board LED is connected to on your Arduino
  model, check the Technical Specs of your board at:
  https://www.arduino.cc/en/Main/Products

  modified 8 May 2014
  by Scott Fitzgerald
  modified 2 Sep 2016
  by Arturo Guadalupi
  modified 8 Sep 2016
  by Colby Newman

  This example code is in the public domain.

  http://www.arduino.cc/en/Tutorial/Blink
*/

// the setup function runs once when you press reset or power the board
void setup() {
  // initialize digital pin LED_BUILTIN as an output.
  pinMode(LED_BUILTIN, OUTPUT);
}

// the loop function runs over and over again forever
void loop() {
  digitalWrite(LED_BUILTIN, HIGH);   // turn the LED on (HIGH is the voltage level)
  delay(1000);                       // wait for a second
  digitalWrite(LED_BUILTIN, LOW);    // turn the LED off by making the voltage LOW
  delay(1000);                       // wait for a second
}
  2. Now, let’s upload our sketch to the Arduino. Go to the sketch menu and choose upload.

If you see an amber-colored led on your Arduino flashing once every second, then you’ve successfully completed this lab. Redo this lab if anything has gone wrong.

Summary

You’ve installed the software, connected the Arduino board to the Arduino software, and configured the software to transfer sketches to your Arduino board.
